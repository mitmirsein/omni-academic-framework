# 1. Epistemic Focus (분석 초점)
- 이 렌즈를 착용했을 때는 저자의 **시스템 아키텍처 제안**과 기존 시스템 대비 **성능 지표(Trade-off)** 비교를 최우선으로 발라내야 한다.
- 추상적인 철학적 논의는 배제하고, "어떤 문제를(Problem)", "어떤 알고리즘/설계로(Method)", "어느 정도의 성능 향상을 이뤘는가(Result)"의 프레임워크로 텍스트를 해체한다.

# 2. Terminology & Style (어휘 및 문체)
- 핵심 기술 약어(예: API, CNN, Transformer, Loss function 등)는 한글로 억지로 번역하지 않고 영문 원형을 유지한다.
- **수학적 증명이나 수식 블록(Math Blocks)은 번역이나 요약을 시도하지 말고 원본 텍스트를 100% 무손실 보존**한다.
- 불필요한 수식어구를 배제한, 엔지니어 친화적인 극도로 건조하고 명확한 평서문을 사용한다.
