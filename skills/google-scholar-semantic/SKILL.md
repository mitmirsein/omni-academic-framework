# 🎓 Google Scholar Semantic (Browser) Recon Skill

> **Identity**: 브라우저 기반 구글 스콜라 시맨틱 검색망 및 인지 엔진 (RISE Protocol)
> **Core Tool**: `scripts/scholar_runner.py` -> `agents/stealth_browser.py`
> **Target**: [Google Scholar Labs (Beta)](https://scholar.google.com/scholar_labs/search?hl=ko) / Google Scholar Web
> **Engine**: `stealth-browser` (for Labs interface) / JSONL parser for `easy-review-system`

## 📌 개요 (Overview)
`google-scholar-semantic` 스킬은 **Playwright CDP (Stealth Browser)**를 가동하여 실제 구글 스콜라 웹페이지를 탐색하고, 인간과 동일한 방식으로 데이터를 읽어 들이는 '시맨틱 정찰망'입니다.

> **⚠️ 주의 (Semantic Scholar API와의 차이점)**:
> 빠르고 규격화된 메타데이터(JSON/API) 추출이 필요할 때는 **`semantic-scholar` (시맨틱스콜라)** 스킬을 사용하십시오. 본 스킬은 사용자가 인증한 브라우저 세션에서 Google Scholar Labs 결과를 사람이 검토하듯 수집하고, 비정형 결과를 EvidencePack용 JSONL로 정리하는 데 특화되어 있습니다.

## 🛠️ Operational Protocol (운용 규약)

1.  **Target URL**: `https://scholar.google.com/scholar_labs` (AI-assisted results)
2.  **Wait Protocol (40초 룰)**: 쿼리 실행 후 결과가 완전히 렌더링될 때까지 **최소 40초** 대기. (Labs 엔진의 심층 분석 결과를 모두 확보하기 위함)
3.  **Deep Extraction (인용 정보 동시 추출)**: 
    - 각 검색 결과 하단의 **'인용(Cite)'** 아이콘을 순차적으로 클릭.
    - 팝업창에 나타나는 formatted citation rows 및 BibTeX/EndNote/RefMan/RefWorks 등 citation links를 추출하여 데이터셋에 병합.
    - 기본값은 `--citation-depth all`이며, `--max-results`로 파싱할 각 레퍼런스에는 citation extraction을 시도해야 한다.
    - 검색 화면 스크랩 시 이 과정을 생략하는 것은 **불완전 정찰**로 간주함.
4.  **Session Quota Control**: 한 브라우저 세션당 **최대 4개 쿼리**로 제한. 5개째부터는 반드시 브라우저 세션을 재시작하여 쿼터 잠금을 방지. Runner는 `--max-queries-per-session` 값을 4 이하로 강제한다.
5.  **Output Integrity**: 수집된 데이터는 각 레퍼런스별 `citation`, `citation_variants`, `citation_links`, `citation_status`를 포함해야 하며, 최종 리포트의 참고문헌 SSOT로 사용함. JSONL 필드 정의는 [output-schema.md](./references/output-schema.md)를 따른다.
6.  **[Question-First Querying]**: Scholar Labs 쿼리는 키워드 나열이 아니라 완결된 자연어 문장으로 작성한다. 가능하면 의문문을 사용한다.
   - Bad: `LLM summary grounding evaluation`
   - Good: `How can researchers determine whether an abstractive summary generated by an LLM is grounded?`
7.  **[TRE Expansion Is Opt-In]**: `data/tre_terms.csv` 확장은 신학/성서학 도메인에서만 사용한다. 범용 Scholar Labs 질문은 기본적으로 TRE 확장 없이 질문형 문장만 정규화한다. 필요 시 `--tre-expand` 또는 `--domain theology`를 명시한다.
8.  **[Evidence-Driven]**: 최소 10개 이상의 유의미한 학술 결과를 확보하며, 'Cited by'와 인용 네트워크를 통해 학술적 위계를 분석 보고한다.
9.  **[JSONL Injection]**: Labs HTML 또는 비정형 텍스트는 `scholar_labs.jsonl`로 변환하여 `easy-review-system`의 EvidencePack에 주입한다.

## 2. Dynamic Workflow
본 연구 전 **분석 함정(Gotchas)**과 **실행 옵션**을 먼저 점검합니다.

### Phase 0: Setup & Guardrail

- **Verify Options**: 연구 레벨(Level 1~3), `--domain`, `--tre-expand`, `--max-results`, `--citation-depth`, `--max-queries-per-session`을 명확히 정합니다. 반복 실행은 `scripts/config.example.json`을 복사해 `--config`로 고정하고, 구조는 `scripts/config.schema.json`을 따릅니다.
- **Gotcha Check**: [gotchas.md](./references/gotchas.md)를 읽고 단일 언어 고립 및 확증 편향을 방지합니다.

### Phase 1: High-Precision Query
학문적 수준을 설정하고 글로벌 쿼리를 생성합니다. 레벨 정의는 [depth-definitions.md](./references/depth-definitions.md)를 참조하십시오.

### Phase 2: Execution (Runner)
`scripts/scholar_runner.py`를 실행하여 교차 검색을 수행합니다. 로직 지침은 [rise-logic.md](./references/rise-logic.md)를 참조하십시오.

```bash
uv run --extra scholar-browser python skills/google-scholar-semantic/scripts/scholar_runner.py \
  --query-file Evidence/SESSION/QuerySet.json \
  --output-dir Evidence/SESSION/google_scholar_labs \
  --jsonl Evidence/SESSION/google_scholar_labs/scholar_labs.jsonl \
  --max-queries-per-session 4 \
  --citation-depth all \
  --max-results 10
```

설정 파일로 실행할 때:

```bash
uv run --extra scholar-browser python skills/google-scholar-semantic/scripts/scholar_runner.py \
  --config skills/google-scholar-semantic/scripts/config.example.json
```

브라우저 없이 경량 검증만 수행할 때:

```bash
uv run --extra scholar-browser python skills/google-scholar-semantic/scripts/scholar_runner.py --self-test
```

이미 저장된 Labs HTML/텍스트만 파싱할 때:

```bash
uv run --extra scholar-browser python skills/google-scholar-semantic/scripts/scholar_runner.py \
  --html scholar_result_1.html \
  --output-dir /tmp/scholar_labs_parse \
  --jsonl /tmp/scholar_labs_parse/scholar_labs.jsonl
```

### Phase 3: Synthesis & Report
논문들 간의 논쟁 구도를 포착하여 리포트를 발행합니다. 결과는 `Research_Report_...md`와 `scholar_labs_*.jsonl`에 저장됩니다. `citation_status`가 `ok`가 아닌 레퍼런스는 보고서에서 누락/재시도 후보로 표시합니다.
