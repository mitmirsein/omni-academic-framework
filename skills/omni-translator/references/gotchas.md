# 📚 Theology Translation: Gotchas & Anti-Patterns

신학 문서 번역 시 에이전트가 흔히 범하는 실수와 이를 방지하기 위한 지침입니다.

## 1. Terminological Footguns (용어적 함정)
- **교파적 혼용 (Standard: Korean Protestant)**: 한국 개신교(교단 연합의 보편적 기준) 용어를 사용하십시오. 가톨릭의 '성사' 대신 '성례', '미사' 대신 '예배', '요한' 대신 '요한복음/사도 요한' 등 개역개정판 성경에 기초한 명칭을 사용해야 합니다.
- **학술적 중립성 상실**: 번역자의 사적인 신학 입장이 개입되어 원문의 뉘앙스를 왜곡하는 경우를 극도로 경계하십시오. 단, 번역 용어는 '한국 개신교 표준'을 따릅니다.
- **사전적 정의의 한계**: `δικαιοσύνη`를 단순히 '의(Righteousness)'로만 번역하지 말고, 바울 신학적 문맥(칭의, 관계적 의 등)을 상기하십시오.
- **시대 착오적 번역 (Anachronism)**: 16세기 텍스트에 현대적 사회과학 용어를 사용하는 실수를 방지하십시오 (Author Mode에서 특히 주의).
- **일관성 붕괴**: 동일 문서 내에서 핵심 개념(예: 'Justification')의 번역어가 '칭의'와 '의인'으로 뒤섞이지 않게 하십시오.

## 2. Structural Failures (구조적 실패)
- **문맥 단절**: 청크(Chunk) 단위로 번역할 때, 앞뒤 청크와의 용어 및 어조 일관성을 놓치는 경우가 많습니다. 반드시 이전 턴의 마지막 Anchor를 확인하십시오.
- **과도한 의역 (Over-Interpretation)**: 신학자의 고유한 문체와 만연체 구조를 지나치게 평이하게 '해설'하지 마십시오. 번역은 주석이 아닙니다.
- **요약의 함정 (The Summary Trap - CRITICAL)**: 번역문이 가독성을 위해 원문의 세부 예시, 부연 설명, 반복 강조를 생략하고 핵심만 전달하는 경우입니다. 이는 **번역 실패**로 간주됩니다. 학술 번역은 모든 정보의 밀도를 100% 보존해야 합니다.

## 3. Agentic Errors (에이전트적 오류)
- **Hallucinated References**: 성경 구절 인용 시 자신이 아는 구절로 대체하지 말고, 원문이 인용한 특정 역본(NRSV, ESV 등)의 의미를 보존하십시오.
- **Review Loop Skip**: `Reviewer` 페르소나가 `Draft`를 'Good'으로만 평가하고 실질적인 비판을 생략하는 경우 -> `Editor`는 반드시 이 점을 지적하십시오.
- **OCR 오인**: 원본이 이미지/PDF일 경우 (Multimodal Mode) OCR 오타가 번역에 그대로 반영되지 않도록 문맥을 철저히 확인하십시오.

## 4. 🔴 Red-Team Adversary Protocol (MANDATORY — Phase 3 주입)

> **경고**: Reviewer는 Phase 3에서 반드시 아래 역할로 전환해야 합니다.
> 이 프롬프트를 실행하지 않으면 Phase 3 완료로 인정하지 않습니다.

```
당신은 이제 번역가의 변호사가 아니라 편집장의 '감사관(Auditor)'입니다.
당신의 유일한 임무는 원문과 번역문을 문장 단위로 대조하여
번역가가 빠뜨리거나 축소한 정보를 찾아내는 것입니다.

### 출력 형식 (반드시 준수):
| ¶ ID | 원문 핵심 문장 | 번역문 대응 문장 | 판정 | 비고 |
|---|---|---|---|---|
| ¶N | "원문..." | "번역..." | ✅/❌ | 누락 설명 |

### 규칙:
- '전체적으로 잘 번역되었습니다'라고만 보고하면 즉시 반려됩니다.
- 최소 3개 이상의 잠재적 문제(완전 누락, 부분 축소, 각주 탈락)를 지적하십시오.
- 지적 사항이 없을 경우 '발견하지 못함'이 아니라 '더 세밀히 검토 중'으로 표기하십시오.
- 이 대조표가 완성된 이후에야 PM은 최종 승인을 검토합니다.
```

## 5. 🚨 Anti-Summary Tripwire (요약 방지 지뢰선)

> **v5.1 신규**: Fidelity Harness의 핵심 방어선.

번역문에서 다음 패턴이 탐지되면 **즉시 HALT** → PM 수동 검토로 전환합니다.

### 탐지 대상 패턴:
```
[CATEGORY A — 직접 요약 표현]
- "~에 대해 설명한다" / "~을(를) 요약하면"
- "~의 요지는" / "핵심은 ~이다" / "결론적으로"
- "(중략)" / "(이하 생략)" / "등등" / "기타"

[CATEGORY B — 메타 해설 (원문에 없는 설명)]
- "저자는 여기서 ~을 주장한다"
- "이 단락은 ~을 다루고 있다"
- "위의 논의를 종합하면"

[CATEGORY C — 정보 압축 징후]
- 원문 3문장 이상이 번역문 1문장으로 축약
- 원문의 구체적 예시/인용이 "~같은 것들"로 일반화
- 각주 번호가 번역문에서 누락
```

### 탐지 시 대응:
1. Red-Team Auditor가 해당 ¶ ID와 패턴을 표로 보고.
2. PM이 원문과 대조하여 **의도적 축소인지 자연스러운 한국어 표현인지** 판정.
3. 의도적 축소로 판정 시 → 해당 ¶ 재번역 지시.
